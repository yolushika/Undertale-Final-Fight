## 2️⃣ 场景管理系统

### 2.1 SceneManager 单例实现

**核心代码：**

```java
public class SceneManager {
    private static SceneManager instance;
    private HashMap<SceneEnum, Scene> scenes = new HashMap<>();
    private Scene currentScene;
    
    // 双重检查锁定的单例模式
    public static SceneManager getInstance() {
        if(instance == null) {
            synchronized(SceneManager.class) {
                if(instance == null) {
                    instance = new SceneManager();
                }
            }
        }
        return instance;
    }
}
```

**设计要点：**
- ✅ 双重检查锁定（Double-Checked Locking）保证线程安全
- ✅ 使用 HashMap 存储所有场景实例
- ✅ `currentScene` 持有当前活动场景

---

### 2.2 场景切换流程

```java
public void switchScene(SceneEnum type, boolean force) {
    if(!force && !shouldSwitch) return;
    if (currentScene != null) {
        currentScene.onExit();  // 退出当前场景
    }
    currentScene = scenes.get(type);
    if (currentScene != null) {
        currentScene.onEnter(); // 进入新场景
    }
    shouldSwitch = false;
}
```

**流程图：**

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  当前场景     │ ──→ │   场景切换    │ ──→ │   新场景      │
│   onExit()   │     │  switchScene │     │   onEnter()  │
│  (清理资源)   │     │              │     │  (初始化)     │
└──────────────┘     └──────────────┘     └──────────────┘
```

## 3️⃣ 状态模式深度解析

### 3.1 为什么需要状态模式？

**重构前（问题代码）：**

```java
// 大型 switch 语句，难以维护
public void handleInput() {
    switch(currentMenuState) {
        case MAIN:
            if (selectedAction == 0) { /* FIGHT */ }
            else if (selectedAction == 1) { /* ACT */ }
            // ... 大量嵌套逻辑
            break;
        // ... 更多状态
```

**问题：**
- ❌ 代码臃肿，一个方法几百行
- ❌ 状态切换逻辑散落各处
- ❌ 新增状态需要修改多处代码

### 3.6 典型状态类分析

#### MainMenuState（主菜单选择）

```java
public class MainMenuState extends AbstractMenuState {
    @Override
    public void handleSelect(MenuStateContext context) {
        MenuState nextState = switch(context.selectedAction) {
            case 0 -> StateFactory.createState(MenuStateType.FIGHT_SELECT_ENEMY);
            case 1 -> StateFactory.createState(MenuStateType.ACT_SELECT_ENEMY);
            case 2 -> StateFactory.createState(MenuStateType.ITEM_SELECT_ITEM);
            case 3 -> StateFactory.createState(MenuStateType.MERCY_SELECT_ENEMY);
            default -> this;
        };
        context.setState(nextState);
    }
}
```

#### ItemSelectItemState（物品选择与滚动）

```java
public class ItemSelectItemState extends AbstractMenuState {
    @Override
    public void handleSelectDown(MenuStateContext context) {
        int itemsPerPage = 4;
        if (context.selectedItem < context.getPlayer().getItemNumber() - 1) {
            context.selectedItem++;
            // 列表滚动逻辑
            if (context.selectedItem >= context.itemListFirstIndex + itemsPerPage) {
                context.itemListFirstIndex++;
            }
        }
    }
    
    @Override
    public void handleSelect(MenuStateContext context) {
        Item item = context.getPlayer().getItemByIndex(context.selectedItem);
        context.getPlayer().heal(item.getHealingAmount());
        context.getPlayer().removeItemByIndex(context.selectedItem);
        context.setState(StateFactory.createState(MenuStateType.ITEM));
    }
}
```

```java
public class BattleFightScene extends Scene {
```

### 5.3 DifficultySelectScene 核心逻辑

```java
public class DifficultySelectScene extends Scene {
    @Override
    public void processInput(boolean[] preKeyStates, boolean[] currKeyStates) {
        if (currKeyStates[GLFW_KEY_Z] && !preKeyStates[GLFW_KEY_Z]) {
            Difficulty[] difficulties = {Difficulty.SIMPLE, Difficulty.NORMAL, Difficulty.HARD};
            DifficultyManager.getInstance().setDifficulty(difficulties[selectedIndex]);
            
            // 选择难度后重置玩家和敌人状态
            Game.getPlayer().resetPlayer();
            Game.getObjectManager().getEnemyManager().resetEnemies();
            
            SceneManager.getInstance().switchScene(SceneEnum.BATTLE_MENU, true);
        }
    }
}
```

## 6️⃣ 设计模式总结

| 设计模式 | 应用位置 | 核心作用 |
|---------|---------|---------|
| **单例模式** | SceneManager, DifficultyManager | 全局唯一实例，统一管理 |
| **工厂模式** | SceneFactory, StateFactory | 集中创建对象，解耦创建逻辑 |
| **状态模式** | MenuState + MenuStateContext | 消除大型switch，状态行为封装 |
| **模板方法模式** | Round 抽象类 | 定义回合骨架，子类实现细节 |
| **观察者模式** | InputObserver | 输入事件分发 |
